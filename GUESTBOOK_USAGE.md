# 留言板使用指南

## 📝 功能概述

博客留言板现已完全激活！您可以在这里发布留言、回复其他用户的评论，与读者进行互动交流。

## ✨ 主要功能

### 🗨️ 发布留言
- **基本信息**：填写昵称和邮箱（必填）
- **留言内容**：输入您想说的话（必填）
- **表单验证**：自动验证邮箱格式和必填项
- **信息保存**：自动记住您的昵称和邮箱

### 🔄 互动回复
- **回复功能**：点击"回复"按钮对任意留言进行回复
- **嵌套显示**：回复以缩进形式展示，形成完整的对话流
- **实时计数**：显示每条留言的回复数量

### 💾 数据存储
- **本地存储**：所有留言数据保存在浏览器本地存储中
- **持久化**：刷新页面后留言数据依然保留
- **隐私安全**：数据不会上传到服务器，完全本地化

### 🎨 界面特色
- **主题适配**：完美支持深色/浅色主题切换
- **响应式设计**：适配各种屏幕尺寸
- **时间显示**：
  - 刚刚发布：显示"刚刚"
  - 近期留言：显示"X分钟前"、"X小时前"、"昨天"
  - 较早留言：显示具体日期

## 📱 访问方式

访问留言板页面：
- 直接访问：`/guestbook`
- 通过导航栏"留言板"链接

## 🛠️ 技术特点

### 前端技术
- **React + TypeScript**：组件化开发，类型安全
- **Next.js**：服务端渲染和客户端交互
- **TailwindCSS**：响应式样式设计

### 数据管理
- **localStorage API**：浏览器本地存储
- **状态管理**：React Hooks状态管理
- **数据验证**：表单验证和邮箱格式检查

### 用户体验
- **加载状态**：提交时显示加载状态
- **错误处理**：友好的错误提示信息
- **表单重置**：发布成功后自动清空输入框

## 📊 数据格式

留言数据使用以下JSON结构存储：

```typescript
interface Message {
  id: string;           // 唯一标识符
  name: string;         // 昵称
  email: string;        // 邮箱
  content: string;      // 留言内容
  timestamp: Date;      // 发布时间
  replies: Reply[];     // 回复列表
}

interface Reply {
  id: string;           // 唯一标识符
  messageId: string;    // 所属留言ID
  name: string;         // 回复者昵称
  email: string;        // 回复者邮箱
  content: string;      // 回复内容
  timestamp: Date;      // 回复时间
}
```

### 团队协作

## ⚠️ 注意事项

1. **数据限制**：留言数据仅存储在当前浏览器中，清除浏览器数据会丢失留言
2. **跨设备同步**：不同设备间的留言数据不会自动同步
3. **隐私保护**：请勿在留言中输入敏感个人信息
4. **内容责任**：用户需对留言内容负责，遵守相关法律法规
